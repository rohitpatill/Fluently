# frontendSrcComponentsContext.md — scope: `frontend/src/components/`

Parent: [../frontendSrcContext.md](../frontendSrcContext.md).

| File | Purpose |
|---|---|
| `Login.jsx` | Auth gate screen (shown by App when `useMe` is unauthenticated). Single "Continue with Google" button (official multi-color G glyph as inline SVG) → `api.loginWithGoogle()` (full-page redirect into the backend OAuth flow). Fluently design language (gradient bg, card, Newsreader-italic subcopy). Reads `?auth_error=1` from the URL to show a gentle inline "try again". No signup page — Google auto-creates the account on first login. |
| `Onboarding.jsx` | 3-step first-run flow. Step 1: persona name + relation chips (5 presets + custom input) + personality textarea ⇒ `PUT /api/memory/persona/form`. Step 2: persona speech-bubble intro, user name + optional about — does NOT submit yet, just advances (so the about-text is only LLM-structured AFTER a key exists). Step 3: **`BrainStep`** ("How smart should I be?", exported for reuse) — paste Gemini key → **Verify** (`api.setModelKey(key,'swift')`: verifies AND stores) → tier cards appear (progressive reveal) → pick Swift/Sage → Continue appears → `finishWithModel` sets the chosen tier (`api.setModelTier`) then `api.submitOnboarding(name, about)`. Shows the "{persona} is getting to know you…" pulsing-avatar overlay during that finish. `onComplete` ⇒ App switches view to 'words' + invalidates ['memory']/['me']. |
| `Rail.jsx` | Responsive nav: left rail (68px) on `md+`, fixed bottom bar on mobile (with text labels). EXACTLY 3 tabs (Chat/Words/Memory — never add a 4th without explicit instruction) + Settings. Desktop-only Google-avatar button (`me.picture`, `referrerPolicy="no-referrer"`, initial fallback) opens Settings. Takes `personaAvatar` (active persona's image) for its persona avatar. `hidden` prop (from App's `useKeyboardInset`) hides the mobile bar while the soft keyboard is open (`hidden md:flex` — desktop rail always shows). |
| `Chat.jsx` | The heart. Takes `personaName`/`personaAvatar`/`personaId` (props from App); `useConversations(personaId)` reads the **persona-scoped** conversation cache. An effect keeps `activeId` consistent with the current list — auto-selects the newest, and CLEARS the selection (+ optimistic bubbles) if the open chat is no longer in the list (persona switched or chat deleted), so the main pane never shows another persona's / a deleted chat (guards on `isLoading` AND `isFetching` — the latter so a just-created chat isn't judged "stale" against the pre-refetch list and bounce the user back to the old chat). All `PersonaAvatar`s pass `avatarUrl={personaAvatar}`. Conversations live in a desktop sidebar, but on mobile move into a left slide-out drawer opened from the compact header (menu/persona area); drawer includes persona header, search, new chat, and thread list. New chat (`newChat`) creates with `suggest_topics:false` → NO LLM wait, lands in the new empty chat instantly; thread delete = two-step inline "sure?" (desktop: on hover; mobile drawer: always-visible trash icon since phones have no hover). Main: header (persona avatar/name/online dot, 30s clock pill), empty-conversation state = a **two-choice landing** (time-aware greeting + composer ready to type): "Let {persona} start ✦" ⇒ opener endpoint, "Suggest topics" ⇒ `suggestTopics()` (on-demand `POST .../{id}/topics`, `topicsLoading` shows skeleton cards → topic cards) ⇒ pick a card = PATCH category + auto-send "Let's talk about: {title}". Buttons stack on phone, row on ≥sm, messages (assistant = markdown bubble left, user = accent bubble right), optimistic pending user bubble, typing indicator. Scoring chips (`ScoringChip`/`ScoringChips`): shown from `chipsByMessage` session state right after a send, else from the message's persisted `word_events` (so they survive refresh); each chip shows the WORD it scored + delta and is click-to-expand/collapse (chevron flips) to reveal the full judge note; green perfect / amber awkward+notes / red wrong+notes; chip is `max-w-full` + note shrinks so it never overflows the bubble width on mobile. Developer mode (`useDevMode`): when ON, `ToolCalls` renders under each assistant reply — an outer "N tool calls" pill expands the group, each `ToolCallCard` independently collapses to show that call's name + Input (args) + Output; reads the message's stored `tool_calls`. Both bubbles run message text through `<WordGloss>` (see its row) so tracked words are tappable-for-meaning inline. Composer: auto-sizing textarea, Enter sends, **text messages capped at `MAX_MESSAGE_CHARS`** (`import.meta.env.VITE_MAX_MESSAGE_CHARS`, default 1200; textarea `maxLength` + `send()` guard + a counter that surfaces only in the last 10%, red at cap; voice is uncapped), + a **mic button** (shown when `useVoiceStatus().available`) that opens `<VoiceOverlay>` for real-time voice; on close it invalidates messages/conversations/words/dashboard/memory (the voice turns were persisted server-side). Mobile keyboard fix: `useKeyboardInset` pads the main column bottom by the keyboard height so the composer sits directly above the soft keyboard (WhatsApp-style); desktop unaffected (inset always 0). |
| `VoiceOverlay.jsx` | Full-screen VOICE MODE experience (opened from Chat's mic button). Blurred backdrop over the app; the persona avatar centered, **breathing while listening / pulsing with expanding rings while speaking** (drives from `useVoiceSession`'s `speaking`/`listening`). Shows a status line ("Listening…"/"{persona} is speaking…"), the current turn's live transcript (persona reply + the user's own words), floating **word-score pops** (`ScorePop`, auto-dismiss ~2.6s — the USP, shown to everyone regardless of Developer mode), and a red stop button (X / Escape / stop all end the session, which persists the final turn, then calls `onClose`). Purely presentational — all logic in `useVoiceSession`. |
| `WordGloss.jsx` | In-chat word gloss (used by `Chat.jsx`, both bubbles). `WordGlossProvider` (takes the word list, builds a normalized single-token→word map; renders the ONE shared popover) + `GlossedText` (used as react-markdown `p/strong/em/li` renderers for assistant bubbles AND directly on the plain user-bubble string via `onAccent`). Matched words get a dotted underline whose intensity scales with score (slipping = stronger; near-mastered = faint; white on the accent user bubble). Hover (desktop) / tap (mobile) opens a popover with word + LLM meaning + user note. Popover is **portaled to `<body>`**, positioned against the viewport — a **centered scrollable sheet on mobile** (dimmed backdrop) / **anchored-and-clamped card on desktop** — so it never clips off-screen; closes on backdrop click / Esc / × / re-tap. Matching is exact-word, case-insensitive, punctuation-stripped; phrases skipped. Purely client-side (no API). |
| `Words.jsx` | Stats row (Mastered/Average/Strongest/Slipping-amber from dashboard stats). A **search box** (shown once ≥1 word) client-side filters rows by text substring (clear ×, "no matches" state). Add input (kind=phrase if contains space; "Enriching…" pending state; 409 friendly toast; auto-expands new word). Rows sorted score desc: responsive row/card layout, ScoreBar, trophy at 100. Expanded detail is a single full-width stack: `PersonalNote` at TOP (user's own memory hook — "+ add a personal note" when empty, else the note with a pencil edit button that is always visible on mobile and hover-revealed on desktop; textarea capped at 200 chars w/ counter + Save/Cancel ⇒ `PUT /api/words/{id}/note`), then meaning+register, 2 examples serif-italic, collocation pills, "lower score −10" + two-step remove, then `EventHistory` (collapsible strip at the bottom, collapsed by default, events lazy-loaded only when opened). |
| `SettingsView.jsx` | Settings screen (reached via the Rail gear). `Account` section: `ProfileCard` + **Log out**. **`Personas` section:** renders `<PersonasSettings>` (the compact summary card → Manage overlay). **`Brain` section (`BrainSettings`):** the two `TierCard`s (Swift/Sage) for instant tier switching + an `API key` card with a **Replace key** sub-form. `Developer` section: `ToggleRow` for **Developer mode**. `Data management` section: `DangerCard`s (purge conversations / memories / reset-keep-words / delete-everything) — soft vs hard (type-to-confirm). NOTE: reset/delete-everything now also wipe ALL personas server-side (re-onboard recreates one); delete-everything additionally clears the model key+tier. |
| `Personas.jsx` | **Multi-persona management.** Default export `PersonasSettings` = a compact summary card (active persona avatar+name, companion count, **Manage →**) that opens a full-screen `ManageOverlay` (dimmed backdrop; centered rounded sheet on desktop, full-height bottom sheet on mobile mirroring the conversations drawer; Escape/backdrop close, body-scroll lock; body scrolls internally so any number of personas never disturbs Settings). Inside, `ManagePanel` has a **"Your personas" | "Discover"** segmented toggle. *Your personas*: `PersonaRow`s (avatar, name, Active badge, conversation count; Switch/Edit/Delete — delete is inline two-step showing the thread count, hidden for the last persona, auto-switches active) + "Add a new persona"; create/edit use the self-contained `PersonaForm` (name, relation chips+custom, **gender chips**, **Voice** row → opens a `VoicePicker` modal [voices grouped by gender with tone/pitch descriptors + a "Hear these in Google AI Studio" link; blank ⇒ gender default], personality, speaking style, avatar URL + live avatar preview — mirrors onboarding step 1 but adds gender+voice; kept separate so onboarding is untouched). *Discover*: `DiscoverPanel` reads `usePersonaCatalog`, category sections of circular image+name `DiscoverCard`s (grid `items-start` so expanding one never stretches its row-mates) → tap expands in place (full-row) → description + **Use this persona** (`api.usePersonaFromCatalog` copies it, closes the overlay, lands on Settings with it in Your personas). All actions invalidate ['personas'] (+ ['me']/['memory']/['conversations'] on switch, so the app-wide active companion updates). |
| `Memory.jsx` | 3 pills (About you=identity / Your life=memory / Who {persona} is=persona). Per file: View/Edit segmented toggle left of Save. View (default on open/tab-switch) = rendered markdown via `ReactMarkdown` (`withLineBreaks` helper adds hard breaks so line-oriented entries don't collapse into one paragraph — markdown ignores single `\n`). Edit = raw monospace textarea bound to `raw`, dirty tracking. Save ⇒ `PUT /api/memory/{file}/raw`, "Saved ✓" state, auto-returns to View. Entry count header, window.confirm before discarding unsaved edits on tab switch. NOT a per-line list (deliberate deviation from prototype). |
| `Shared.jsx` | `PersonaAvatar` (gradient circle with serif-italic initial, sizes xs–xl, optional online dot; **optional `avatarUrl`** renders a public image over the gradient with `onError` fallback to the initial — used everywhere the active persona shows), `Spinner`, `FullScreenLoader`, `FullScreenError` (title/message/retry), `ScoreBar` (motion width animation, amber when slipping), **`TierCard`** (one selectable Swift/Sage "brain" card — icon [Zap/Sparkles], name, model id in mono, tagline, price; Sage gets a "More powerful" accent tag; selected = accent border + check; shared by Onboarding's `BrainStep` and Settings' `BrainSettings`). **Skeletons** (for the network latency of the Mongo backend): base `Skeleton` primitive (calm 1.5s gradient-sweep shimmer via `.skeleton-shimmer` / `skeletonSweep` keyframe in index.css, neutral `border-2` base — no new colors) + content-shaped `WordRowSkeleton`, `ThreadItemSkeleton`, `MessageBubbleSkeleton` (alternating via `mine` prop), `StatCardSkeleton`, `MemoryEditorSkeleton`, `DiscoverCardSkeleton` (Discover catalog card), `PersonaRowSkeleton` (Your-personas row); exported `REVEAL` motion preset (`opacity/y` 0.25s, `[0.2,0.8,0.2,1]` ease) used to fade real content in over the skeletons. |

Loading UX: page/list loads show content-shaped SKELETONS, not bare spinners — `Words.jsx` (4 `StatCardSkeleton` + 6 `WordRowSkeleton`), `Chat.jsx` (4 `ThreadItemSkeleton` sidebar, 4 alternating `MessageBubbleSkeleton` in the message area), `Memory.jsx` (`MemoryEditorSkeleton`). Real rows fade in via `REVEAL`. Inline ACTION spinners (Saving…/Enriching…/Opening…, new-chat button, event-history) are left as spinners — those are action feedback, not page loads.

Conventions: mutations via `useMutation` + `api.*` + targeted `invalidateQueries`; toasts for every success/failure; two-step inline confirms (2.5s timeout) instead of modal dialogs; all animation via motion or the @theme keyframes; entity ids from the API are opaque STRINGS (Mongo ObjectId hex) — never assume numeric.
