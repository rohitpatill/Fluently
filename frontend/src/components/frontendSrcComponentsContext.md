# frontendSrcComponentsContext.md — scope: `frontend/src/components/`

Parent: [../frontendSrcContext.md](../frontendSrcContext.md).

| File | Purpose |
|---|---|
| `Onboarding.jsx` | 2-step first-run flow. Step 1: persona name + relation chips (5 presets + custom input) + personality textarea ⇒ `PUT /api/memory/persona/form`. Step 2: persona speech-bubble intro, user name + optional about ⇒ `api.submitOnboarding(name, about)` (backend LLM-structures the about-text across identity/memory/persona). Shows a "{persona} is getting to know you…" pulsing-avatar overlay while that call runs (only when about-text is non-empty). Calls `onComplete` ⇒ App invalidates ['memory']. |
| `Rail.jsx` | 68px left nav. EXACTLY 3 tabs (Chat/Words/Memory — never add a 4th without explicit instruction). Persona avatar top, user initial bottom. |
| `Chat.jsx` | The heart. Threads sidebar (search filter, two-step inline delete "sure?", + new chat stores returned topics in `topicsByConv` state). Main: header (persona avatar/name/online dot, 30s clock pill), empty-conversation state (time-aware greeting, topic cards ⇒ PATCH category + auto-send "Let's talk about: {title}", "Let {persona} start ✦" ⇒ opener endpoint), messages (assistant = markdown bubble left, user = accent bubble right), optimistic pending user bubble, typing indicator. Scoring chips (`ScoringChip`/`ScoringChips`): shown from `chipsByMessage` session state right after a send, else from the message's persisted `word_events` (so they survive refresh); each chip is click-to-expand/collapse (chevron flips) to reveal the full judge note; green perfect / amber awkward+notes / red wrong+notes. Developer mode (`useDevMode`): when ON, `ToolCalls` renders under each assistant reply — an outer "N tool calls" pill expands the group, each `ToolCallCard` independently collapses to show that call's name + Input (args) + Output; reads the message's stored `tool_calls`. Composer: auto-sizing textarea, Enter sends. |
| `Words.jsx` | Stats row (Mastered/Average/Strongest/Slipping-amber from dashboard stats). Add input (kind=phrase if contains space; "Enriching…" pending state; 409 friendly toast; auto-expands new word). Rows sorted score desc: ScoreBar, trophy at 100. Expanded detail is a single full-width stack: `PersonalNote` at TOP (user's own memory hook — "+ add a personal note" when empty, else the note with a hover pencil to edit; textarea + Save/Cancel ⇒ `PUT /api/words/{id}/note`), then meaning+register, 2 examples serif-italic, collocation pills, "lower score −10" + two-step remove, then `EventHistory` (collapsible strip at the bottom, collapsed by default, events lazy-loaded only when opened). |
| `SettingsView.jsx` | Settings screen (reached via the Rail gear). `Developer` section with a `ToggleRow` for **Developer mode** (`useDevMode`, localStorage, off by default — surfaces agent tool calls in Chat). `Data management` section: `DangerCard`s for hard-delete flows (purge conversations / memories / reset-keep-words / delete-everything) — soft (accent-outline confirm) vs hard (type-to-confirm word). |
| `Memory.jsx` | 3 pills (About you=identity / Your life=memory / Who {persona} is=persona). Per file: View/Edit segmented toggle left of Save. View (default on open/tab-switch) = rendered markdown via `ReactMarkdown` (`withLineBreaks` helper adds hard breaks so line-oriented entries don't collapse into one paragraph — markdown ignores single `\n`). Edit = raw monospace textarea bound to `raw`, dirty tracking. Save ⇒ `PUT /api/memory/{file}/raw`, "Saved ✓" state, auto-returns to View. Entry count header, window.confirm before discarding unsaved edits on tab switch. NOT a per-line list (deliberate deviation from prototype). |
| `Shared.jsx` | `PersonaAvatar` (gradient circle, serif-italic initial, sizes xs–xl, optional online dot), `Spinner`, `FullScreenLoader`, `FullScreenError` (title/message/retry), `ScoreBar` (motion width animation, amber when slipping). **Skeletons** (for the network latency of the Mongo backend): base `Skeleton` primitive (calm 1.5s gradient-sweep shimmer via `.skeleton-shimmer` / `skeletonSweep` keyframe in index.css, neutral `border-2` base — no new colors) + content-shaped `WordRowSkeleton`, `ThreadItemSkeleton`, `MessageBubbleSkeleton` (alternating via `mine` prop), `StatCardSkeleton`, `MemoryEditorSkeleton`; exported `REVEAL` motion preset (`opacity/y` 0.25s, `[0.2,0.8,0.2,1]` ease) used to fade real content in over the skeletons. |

Loading UX: page/list loads show content-shaped SKELETONS, not bare spinners — `Words.jsx` (4 `StatCardSkeleton` + 6 `WordRowSkeleton`), `Chat.jsx` (4 `ThreadItemSkeleton` sidebar, 4 alternating `MessageBubbleSkeleton` in the message area), `Memory.jsx` (`MemoryEditorSkeleton`). Real rows fade in via `REVEAL`. Inline ACTION spinners (Saving…/Enriching…/Opening…, new-chat button, event-history) are left as spinners — those are action feedback, not page loads.

Conventions: mutations via `useMutation` + `api.*` + targeted `invalidateQueries`; toasts for every success/failure; two-step inline confirms (2.5s timeout) instead of modal dialogs; all animation via motion or the @theme keyframes; entity ids from the API are opaque STRINGS (Mongo ObjectId hex) — never assume numeric.
