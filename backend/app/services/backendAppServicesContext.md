# backendAppServicesContext.md ‚Äî scope: `backend/app/services/`

All business logic. Parent: [../backendAppContext.md](../backendAppContext.md).

| File | Purpose & key functions |
|---|---|
| `auth_service.py` | Google OAuth + JWT session ‚Äî pure, framework-agnostic helpers (no FastAPI types; unit-testable). `new_state_and_nonce`, `sign_state`/`unsign_state` (itsdangerous `TimestampSigner`, 300s TTL ‚Äî the stateless CSRF/replay guard), `build_google_auth_url` (scope `openid email profile`, `prompt=select_account`), `exchange_code_for_tokens` (httpx POST to Google's token endpoint), `verify_google_id_token` (`google-auth` checks signature/exp/aud; we ALSO assert `iss`, `nonce`, `email_verified`), `mint_session_jwt`/`decode_session_jwt` (PyJWT HS256, algorithm PINNED). `AuthError` on any failure. Stores NO Google tokens ‚Äî reads identity from the ID token and discards. |
| `llm_service.py` | Model factory via LangChain `init_chat_model`, now **per-user key-aware**. `get_chat_model(provider, model, api_key=None, **kw)`, `get_judge_model(...)` (temp 0 default), `get_utility_model(...)` ‚Äî `api_key` is passed straight into `init_chat_model` (NO `os.environ` mutation, so concurrent users' keys never clobber). Providers: openai / anthropic / google_genai (Gemini MUST use explicit `google_genai`). |
| `crypto_service.py` | Fernet symmetric encryption for users' bring-your-own API keys. `encrypt(plaintext)->ciphertext`, `decrypt(ciphertext)->plaintext`; `CryptoError` on misconfig/tamper. Key from `settings.encryption_key` (`.env` `ENCRYPTION_KEY`, generate via `Fernet.generate_key()`) ‚Äî NEVER stored in the DB. The single seam that does key crypto. |
| `model_service.py` | Per-user model resolution. `tiers_public()` (Swift/Sage catalogue for the UI, no secrets), `resolve_for_user(user_id)->ResolvedModel{provider,model,api_key,tier}` (loads user, decrypts key, maps tier‚Üímodel id; raises `NoModelConfigured` if unset), `verify_key(api_key, tier)->bool` (one throwaway `invoke("hi")` with `max_output_tokens=1`, never raises). Tier‚Üímodel map lives in `config.MODEL_TIERS` (swift=gemini-3.1-flash-lite, sage=gemini-3.5-flash). |
| `chat_service.py` | Core turn engine. `run_agent_turn(conversation, user_content?, provider?, model?, extra_instruction?)` ‚Äî NO db arg; persists via `repo`. user_content=None ‚áí opener flow. `extra_instruction` (e.g. the opener directive) is sent as a HumanMessage, NOT appended to the system prompt ‚Äî keeps `contents` non-empty so Gemini doesn't reject the opener with `contents are required.`. Loads messages via `repo.load_messages`. `_history_messages` rebuilds AIMessage(tool_calls)+ToolMessage pairs with ORIGINAL ids (providers reject mismatches). `_flatten_content` ‚Äî Gemini returns content as block LIST; always flatten. Tool loop max 6 iterations; tool errors returned to model, never raised. `_maybe_set_title` after first exchange (failure swallowed). **Resolves the per-user model** via `model_service.resolve_for_user(conversation.user_id)` and passes its key+model into `get_chat_model` and the title `get_utility_model` (the explicit `provider`/`model_name` args still override if given). |
| `judge_service.py` | `judge_user_message(conversation_id, user_message_id, user_id=‚Ä¶)` ‚Äî NO db arg. Judges ALL tracked words (via `repo.list_words`, not just conversation targets) with structured output (`JudgeResult`), last 6 messages as context, maps classifications to scoring events. Resolves the per-user model (`resolve_for_user(user_id)`) for the judge call. Any exception ‚áí returns [] (never breaks chat). |
| `scoring_service.py` | SINGLE SOURCE OF TRUTH for scores. `apply_event(word, event_type, ...)` ‚Äî NO db arg; persists via `repo.save_word`/`repo.insert_event`. Matrix: perfect_unprompted +5, perfect_prompted +3, awkward +1, wrong ‚àí2, passive +0.5, manual=delta. NO daily cap ‚Äî gains apply in full (a word can go 0‚Üí100 in one day); only clamp is 0‚Äì100. `apply_decay(word)` (lazy, ‚àí1/week after 14 idle days, called on words list reads). `pick_target_words(limit?, user_id=‚Ä¶)` (lowest score, then least recently used; excludes score 100). |
| `prompt_builder.py` | `build_system_prompt(conversation)` ‚Äî NO db arg; reads memory files via `memory_service` (Mongo) and target words via `repo.get_words_by_ids`. Assembles CHAT_SYSTEM_TEMPLATE: persona block (from persona file, fallback if unset) ‚Üí identity ‚Üí memory ‚Üí target words with [id=‚Ä¶] + scores (+ the word's `register` and the user's own `memory hook` note when set) ‚Üí rich TIME block (`_time_block`) ‚Üí category. `_part_of_day(hour)` ‚Üí morning/midday/afternoon/evening/late night (no season/weather ‚Äî deliberately not inferred, see prompts.py). |
| `memory_service.py` | Pure free-text markdown engine for identity/memory/persona. **identity & memory** are the SHARED human files (one `memory_files` doc each per user). **persona is PER-PERSONA**: the `persona` "file" transparently routes to the user's ACTIVE persona's `content` (in the `personas` collection), so callers still say `read_file("persona", uid)` unchanged. `_active_persona(uid)` resolves the active persona (self-heals: migrates a legacy single `persona` memory_files doc into a Persona row, or creates an empty one, and repairs a stale active pointer). `active_persona_id(uid)` (public ‚Äî used to stamp/scope conversations). `_set_content` is the single write seam (persona‚Üíactive persona doc, shared‚Üímemory_files). Same read-modify-write logic; NO ids/stamps. `normalize_file`, `append`, `edit(old,new,replace_all)`, `parse_lines`, `read_file`, `write_raw`, `reset_file`, `ensure_files` (bootstraps shared files + guarantees ‚â•1 persona). `build_persona_content(form, existing_raw="")` composes persona markdown preserving `## Relationship memories`; `set_persona(form, uid, persona_id?)` targets a specific persona (edit/create) or the active one (onboarding); `parse_persona_fields(content)` extracts name/relation/gender/personality/speaking_style for the persona cards. |
| `search_service.py` | `search(query, mode=bm25|regex, conversation_id?, n_before, n_after, full_conversation, max_results, exclude_conversation_id?, user_id=‚Ä¶, persona_id?)` ‚Äî NO db arg; loads messages via `repo.all_messages` (which, given `persona_id`, limits to that persona's conversations). BM25 (rank_bm25) / regex ranking still done IN PYTHON (Mongo is only the store), context windows, overlapping-window dedup per conversation. `format_hits_for_llm` renders tool output text. NOTE: BM25 IDF is non-positive for a term present in EVERY document, so single-message corpora return no hits ‚Äî a known quirk, not a bug. |
| `topic_service.py` | `suggest_topics(target_words, user_id=‚Ä¶, persona_id?)` (structured `TopicList`; uses identity+memory+`repo.recent_conversations` titles **scoped to the active persona**), `enrich_word(text, kind, user_id=‚Ä¶)` (structured `WordEnrichment`), `structure_onboarding_info(raw_about, persona_name, today, user_id=‚Ä¶)` (structured `OnboardingFacts{identity,memory,persona}`; `None` on LLM failure). ALL take `user_id` and resolve the per-user model (`resolve_for_user`) inside their try. All return empty/None on any LLM failure. |
| `persona_catalog.py` | Curated public-persona "Discover" catalog. Reads `app/assets/personas.json` (NOT the DB ‚Äî identical for all users, saves free-tier Mongo) and caches via `lru_cache`. `categories()` (grouped list for the UI), `get_entry(catalog_id)` (one figure by stable id). Choosing one COPIES it into the user's `personas` (see `routers/personas.py` `/catalog/{id}/use`); the JSON is read-only. `avatar_url`s are filled manually in the JSON; `description` seeds the copy's Personality. |
| `agent_tools.py` | `build_tools(current_conversation_id, user_id=‚Ä¶, persona_id?)` ‚Üí LangChain @tool list built per-request (closures over `user_id`+`persona_id`, NOT a db session): **`memory_update(file, action, text?, old_string?, new_string?, replace_all?)`** ‚Äî single tool, `append`/`edit` (empty new_string deletes); constrained params are `Literal` enums so the model can't send junk ‚Äî `file` ‚àà identity|memory|persona (still defensively `normalize_file`'d), `action` ‚àà append|edit; outputs use bare names, no `.md`; schema descriptions carry subject-free/date rules; `search_conversations` (excludes current conversation; **scoped to `persona_id`** so a companion only recalls chats it had); `adjust_word_score(word_id: str, delta, reason)` ‚Üí manual event via `repo.get_word` + `scoring_service.apply_event`. Tool outputs stored on assistant Message.tool_calls (surfaced in UI Developer mode). |

Iron rules: services NEVER touch pymongo directly ‚Äî all persistence goes through `app/repo.py` (the single swappable data layer). Every LLM call is scoped to the calling user's OWN key + tier ‚Äî resolve via `model_service.resolve_for_user(user_id)` and pass its `provider/model/api_key` into the factory; NEVER call a factory with the app's own key. User keys are stored ENCRYPTED (`crypto_service`) ‚Äî never log or return plaintext. LLM calls in judge/topics/enrichment/title/onboarding-structuring are best-effort (swallow failures). Never change scoring outside scoring_service.py. New agent tools go in agent_tools.py + get mentioned in prompts.py tool rules. Memory files are pure free-text (no IDs/stamps) ‚Äî dates belong in the text ONLY for time-bound facts, written by the agent itself, resolved as absolute dates against the TIME block. identity/memory are SHARED across a user's personas; the `persona` file is PER-PERSONA (routes to the active persona) ‚Äî a user may keep several companions and switch; threads + conversation-search are persona-scoped, but words/scores/identity/memory are shared. `config.py`'s `user_timezone` (default Asia/Kolkata) drives all temporal computation.
