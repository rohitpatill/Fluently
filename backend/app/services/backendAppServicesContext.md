# backendAppServicesContext.md — scope: `backend/app/services/`

All business logic. Parent: [../backendAppContext.md](../backendAppContext.md).

| File | Purpose & key functions |
|---|---|
| `auth_service.py` | Google OAuth + JWT session — pure, framework-agnostic helpers (no FastAPI types; unit-testable). `new_state_and_nonce`, `sign_state`/`unsign_state` (itsdangerous `TimestampSigner`, 300s TTL — the stateless CSRF/replay guard), `build_google_auth_url` (scope `openid email profile`, `prompt=select_account`), `exchange_code_for_tokens` (httpx POST to Google's token endpoint), `verify_google_id_token` (`google-auth` checks signature/exp/aud; we ALSO assert `iss`, `nonce`, `email_verified`), `mint_session_jwt`/`decode_session_jwt` (PyJWT HS256, algorithm PINNED). `AuthError` on any failure. Stores NO Google tokens — reads identity from the ID token and discards. |
| `llm_service.py` | Model factory via LangChain `init_chat_model`. `get_chat_model(provider?, model?)`, `get_judge_model()` (temp 0), `get_utility_model()`. Providers: openai / anthropic / google_genai (Gemini MUST use explicit `google_genai`). Pushes .env keys into os.environ. |
| `chat_service.py` | Core turn engine. `run_agent_turn(conversation, user_content?, provider?, model?, extra_instruction?)` — NO db arg; persists via `repo`. user_content=None ⇒ opener flow. Loads messages via `repo.load_messages`. `_history_messages` rebuilds AIMessage(tool_calls)+ToolMessage pairs with ORIGINAL ids (providers reject mismatches). `_flatten_content` — Gemini returns content as block LIST; always flatten. Tool loop max 6 iterations; tool errors returned to model, never raised. `_maybe_set_title` after first exchange (failure swallowed). |
| `judge_service.py` | `judge_user_message(conversation_id, user_message_id, user_id=…)` — NO db arg. Judges ALL tracked words (via `repo.list_words`, not just conversation targets) with structured output (`JudgeResult`), last 6 messages as context, maps classifications to scoring events. Any exception ⇒ returns [] (never breaks chat). |
| `scoring_service.py` | SINGLE SOURCE OF TRUTH for scores. `apply_event(word, event_type, ...)` — NO db arg; persists via `repo.save_word`/`repo.insert_event`; daily-cap total via `repo.gained_today`. Matrix: perfect_unprompted +5, perfect_prompted +3, awkward +1, wrong −2, passive +0.5, manual=delta (bypasses cap). Daily cap +10 (positive non-manual only). Clamp 0–100. `apply_decay(word)` (lazy, −1/week after 14 idle days, called on words list reads). `pick_target_words(limit?, user_id=…)` (lowest score, then least recently used; excludes score 100). |
| `prompt_builder.py` | `build_system_prompt(conversation)` — NO db arg; reads memory files via `memory_service` (Mongo) and target words via `repo.get_words_by_ids`. Assembles CHAT_SYSTEM_TEMPLATE: persona block (from persona file, fallback if unset) → identity → memory → target words with [id=…] + scores (+ the word's `register` and the user's own `memory hook` note when set) → rich TIME block (`_time_block`) → category. `_part_of_day(hour)` → morning/midday/afternoon/evening/late night (no season/weather — deliberately not inferred, see prompts.py). |
| `memory_service.py` | Pure free-text markdown engine for identity/memory/persona — now stored as documents in the `memory_files` Mongo collection (via `repo.get_memory_file`/`set_memory_file`), NOT on disk. Same read-modify-write logic (read whole string → mutate → write back). NO ids, NO machine-added stamps. All functions take optional `user_id` (defaults to sentinel). `normalize_file(file)` strips a stray `.md`; `append`, `edit(old_string,new_string,replace_all)` (KeyError if absent; empty new_string deletes + collapses blank gap), `parse_lines` (UI count only), `read_file`, `write_raw`, `set_persona` (rewrites header fields, keeps content under the "## Relationship memories" marker), `ensure_files`, `reset_file`. Headers are heading-only. |
| `search_service.py` | `search(query, mode=bm25|regex, conversation_id?, n_before, n_after, full_conversation, max_results, exclude_conversation_id?, user_id=…)` — NO db arg; loads messages via `repo.all_messages`. BM25 (rank_bm25) / regex ranking still done IN PYTHON (Mongo is only the store), context windows, overlapping-window dedup per conversation. `format_hits_for_llm` renders tool output text. |
| `topic_service.py` | `suggest_topics(target_words, user_id=…)` (structured `TopicList`; uses identity+memory+`repo.recent_conversations` titles), `enrich_word(text, kind)` (structured `WordEnrichment`), `structure_onboarding_info(raw_about, persona_name, today)` (structured `OnboardingFacts{identity,memory,persona}`; `None` on LLM failure). All return empty/None on any LLM failure. |
| `agent_tools.py` | `build_tools(current_conversation_id, user_id=…)` → LangChain @tool list built per-request (closures over `user_id`, NOT a db session): **`memory_update(file, action, text?, old_string?, new_string?, replace_all?)`** — single tool, `append`/`edit` (empty new_string deletes); `file` bare name, defensively `normalize_file`'d; schema descriptions carry subject-free/date rules; `search_conversations` (excludes current conversation); `adjust_word_score(word_id: str, delta, reason)` → manual event via `repo.get_word` + `scoring_service.apply_event`. Tool outputs stored on assistant Message.tool_calls (surfaced in UI Developer mode). |

Iron rules: services NEVER touch pymongo directly — all persistence goes through `app/repo.py` (the single swappable data layer). LLM calls in judge/topics/enrichment/title/onboarding-structuring are best-effort (swallow failures). Never change scoring outside scoring_service.py. New agent tools go in agent_tools.py + get mentioned in prompts.py tool rules. Memory files are pure free-text (no IDs/stamps), stored as Mongo docs — dates belong in the text ONLY for time-bound facts, written by the agent itself, resolved as absolute dates against the TIME block. `config.py`'s `user_timezone` (default Asia/Kolkata) drives all temporal computation.
