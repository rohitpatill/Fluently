# backendAppRoutersContext.md ‚Äî scope: `backend/app/routers/`

HTTP layer only ‚Äî validation + delegation to services/`repo`. Sync `def` routes (FastAPI threadpools them).
Parent: [../backendAppContext.md](../backendAppContext.md).
Routes call `repo` / services directly (no `db` session). Every data route resolves the real user via `user_id: str = Depends(get_current_user)` (from `app/deps.py`) ‚Äî no more hardcoded `DEFAULT_USER_ID`; a missing/invalid session ‚áí 401. **LLM-using routes** (chat send, conversation create, opener) instead depend on `Depends(require_model_configured)` ‚Üí 403 "No model configured" if the user has no key/tier yet (frontend turns this into the model-config gate). Path ids are strings (ObjectId hex); an invalid/absent id yields 404. `voice.py` is the one **WebSocket** router (real-time audio) ‚Äî it authenticates the session cookie itself on the handshake (the HTTP `Depends` chain can't raise on a WS).

| File | Endpoints |
|---|---|
| `auth.py` | Google OAuth + session (prefix `/api/auth`). `GET /google/login` (sets a signed HttpOnly state+nonce cookie, 307 ‚Üí Google consent). `GET /google/callback` (verify state/CSRF ‚Üí exchange code ‚Üí verify ID token ‚Üí `upsert_user_from_google`; first-ever user adopts `default` data; bootstraps the user's memory files; sets the JWT session cookie; redirects to `frontend_url`; any failure ‚Üí `frontend_url/?auth_error=1`, no session set). `GET /me` (current profile + `has_persona` + **`has_key`/`tier`**; 401 if unauthenticated). `POST /logout` (clears the session cookie). All state/session set+delete_cookie calls use `settings.cookie_secure` + `settings.cookie_samesite` (‚Üí `SameSite=None; Secure` when frontend/backend are cross-domain, else `Lax`) so the session survives a split deploy; delete uses the same flags or the cookie won't clear. |
| `model.py` | Bring-your-own-key model config (prefix `/api/model`). `GET /tiers` (Swift/Sage catalogue ‚Äî name/model id/tagline/price, no secrets). `GET /status` (`{has_key, tier}` for the current user ‚Äî NEVER the key). `POST /key` (`{api_key, tier}` ‚Üí `model_service.verify_key` ‚Üí `crypto_service.encrypt` ‚Üí `repo.set_user_key`; 400 if verify fails / unknown tier / empty). `PUT /tier` (`{tier}` ‚Üí switch with the already-stored key; 400 if unknown tier or no key). Powers both onboarding step 3 and Settings' Brain card. |
| `chat.py` | `POST /api/chat/{conversation_id}` ‚Äî full chat turn; returns user_message, assistant_message (with tool_calls), scoring_events. |
| `personas.py` | Multi-persona management (prefix `/api/personas`). `GET ""` (list, each with active flag + conversation_count), `GET /catalog` (curated Discover catalog grouped by category, from `persona_catalog`) + `POST /catalog/{catalog_id}/use` (COPY a catalog figure ‚Äî incl. its hand-picked `voice` ‚Äî into the user's personas; does NOT switch active), `POST ""` (create), `PUT /{id}` (edit identity fields incl. `gender`/`voice`, preserves relationship memories + avatar), `PUT /{id}/avatar` (set/clear public URL), `POST /{id}/activate` (switch active persona), `DELETE /{id}` (delete persona + its conversations/messages; keeps words; refuses the LAST one; auto-switches active if the deleted one was active). Static `/catalog` routes are declared BEFORE `/{persona_id}` so they win. |
| `conversations.py` | `POST /api/conversations` (picks target words + stamps the ACTIVE persona's id via `memory_service.active_persona_id`, optional topic suggestions scoped to that persona ‚Äî the frontend now creates with `suggest_topics:false` so "+" opens instantly with no LLM wait), `POST .../{id}/topics` (**on-demand** topic suggestions for an existing chat ‚Äî the "Suggest topics" button; uses that chat's target words + persona; gated), `GET` list (**persona-scoped ‚Äî only the active persona's threads**)/detail, `GET .../{id}/messages` (each message carries `tool_calls`; USER messages also carry `word_events` ‚Äî that message's scoring events with resolved `word_text` ‚Äî so scoring chips + Developer-mode tool calls survive a refresh), `PATCH .../category` (query param), `POST .../opener` (400 if messages exist), `DELETE`, `POST /api/conversations/search` (BM25/regex, window flags; **scoped to the active persona**). |
| `words.py` | `GET /api/words` (applies lazy decay), `POST` (409 duplicate, LLM enrichment on add), `GET/DELETE /{id}`, `PUT /{id}/note` (user's own memory-hook note; trims, empty clears), `POST /{id}/adjust` (manual event), `GET /{id}/events` (last 100). |
| `memory.py` | `GET /api/memory/{file}` (raw + parsed lines, no ids/timestamps), `POST .../lines` (append), `POST .../edit` (old_string‚Üínew_string, empty new_string deletes; 404 if old_string absent, 400 if empty), `PUT .../{file}/raw` (whole-file overwrite, 400 without "raw" key), `PUT /api/memory/persona/form` (onboarding step 1), `POST /api/memory/onboarding` (onboarding step 2 finish: stores Name deterministically + LLM-structures the free-text "about" box across identity/memory/persona via `topic_service.structure_onboarding_info`; falls back to raw-append to identity on LLM failure). Valid files: identity, memory, persona. |
| `voice.py` | Voice mode ‚Äî real-time duplex audio via Gemini Live (prefix `/api/voice`). **`WS /ws/{conversation_id}`**: cookie-auth on the handshake (`_authenticate`, NOT the HTTP dep ‚Äî WS can't raise HTTPException), then proxies browser‚ÜîGemini through `voice_service.open_session` ‚Äî two relay coroutines (`_browser_to_gemini` mic 16k PCM in; `_gemini_to_browser` audio 24k PCM + transcripts + tool calls out). The receive loop is `while True: async for ‚Ä¶ in session.receive()` (receive() ends each turn ‚Äî the outer loop re-enters it, else the session dies after turn 1). Executes tool calls SERVER-SIDE (`VoiceToolExecutor` ‚Üí `send_tool_response`); a `score_word` result is pushed as a `{"type":"score"}` frame (live animation). `_TurnBuffer` persists EACH completed turn as normal user+assistant `Message`s (assistant carries `tool_calls`; live score events get their `message_id` back-filled to the user msg via `repo.set_event_message_id`) so the voice chat shows in the thread afterward. After the first turn's flush it auto-titles the conversation via `chat_service._maybe_set_title` (same flow as text; run off-loop via `asyncio.to_thread`, self-guards + swallows errors). **REST:** `GET /voices` (voice catalogue for the persona-form picker + `audition_url` to Google AI Studio), `GET /status` (`{available}` = user has a key/tier). |
| `assistant.py` | Fluently in-app VOICE ASSISTANT ‚Äî real-time duplex audio via Gemini Live (prefix `/api/assistant`), SEPARATE from persona voice (`voice.py`): the APP talking to the user for help + a few hands-free actions. Ephemeral ‚Äî **NOTHING persisted, NOTHING scored, no persona, no conversation_id**. **`WS /ws?tab={screen}`**: cookie-auth on handshake (`_authenticate`), opens a Live session via the user's OWN key (`resolve_for_user`) with `assistant_service.build_live_config`, then SPEAKS FIRST (sends `prompts.ASSISTANT_GREETING_INSTRUCTION` as a client turn so it greets the user by name). Two relay loops (mic 16k in / audio 24k + transcripts out); executes `AssistantToolExecutor` tool calls server-side + `send_tool_response`; a successful create/add/switch pushes an `{"type":"action", name, message}` frame ‚Üí live on-screen confirmation + the frontend refreshes the affected caches. **REST:** `GET /status` (`{available}` = user has a key/tier). |
| `dashboard.py` | `GET /api/dashboard/stats` ‚Äî totals, mastered, average, weekly events/gains, top/weakest/slipping words (via `repo` aggregations). |
| `settings.py` | Data-management HARD deletes: `DELETE /api/settings/conversations` (purge conversations+messages, keep words/memories; optional `?persona_id=` scopes the delete to ONE persona's chats, else ALL personas'), `DELETE /api/settings/memories` (reset identity+memory only ‚Äî the shared human files; personas untouched), `POST /api/settings/purge-all` (`{keep_words}` ‚Äî nuke conversations + **ALL personas** (`repo.purge_personas`) + reset identity/memory, optionally words+events; both flows restart at onboarding which recreates one persona; **when `keep_words=False` it's a TRUE full reset that ALSO clears the user's model key+tier via `repo.clear_user_model`** ‚Üí re-onboard the brain; response includes `cleared_model`). All via `repo.purge_*`/`clear_user_model` + `memory_service.reset_file`. |

Conventions: raise `HTTPException` with clear detail; response models from `schemas.py`; no business logic here.
