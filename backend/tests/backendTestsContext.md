# backendTestsContext.md — scope: `backend/tests/`

Permanent pytest suite. Parent: [../backendContext.md](../backendContext.md).
Run: `.venv\Scripts\python run_tests.py [area...]` (area-selectable — no args = all; e.g. `run_tests.py words scoring`; `--list` shows keys) / `--live` (real provider from .env).

| File | Covers |
|---|---|
| `conftest.py` | Points the app at a SEPARATE Mongo database `<MONGODB_DB>_test` (env set BEFORE app import — the real `fluently` DB is never touched). `fresh_db` autouse fixture wipes all collections (incl. `users`) before+after each test and re-ensures indexes. **`client`** fixture = authenticated as fixed `TEST_USER_ID` (overrides `deps.get_current_user`/`get_current_user_obj`, seeds that user + their memory files); existing router tests run unchanged under it. **`anon_client`** = no override (for testing the real auth flow). `seed_user(user_id, ...)` helper (also seeds an ENCRYPTED placeholder key + `model_tier="swift"` so LLM-gated routes pass; the `client` fixture's `User` obj likewise carries key/tier so `require_model_configured` passes). `FakeChatModel`, `FakeStructuredFactory/Model` (canned structured outputs incl. `OnboardingFacts`). `mock_llms` autouse patches all LLM factories AND `resolve_for_user` (on chat/judge/topic modules, returns a fake `ResolvedModel`) unless test marked `@pytest.mark.live`. **`ENCRYPTION_KEY` comes from `.env`** (loaded via `load_dotenv`) — NEVER hardcoded in any test file (they're committed to Git). NOTE: mocked tests still hit REAL Atlas (the `_test` DB) — needs network + valid `MONGODB_URI`. When seeding messages/data directly via `repo`, pass `user_id=TEST_USER_ID` or the authed client won't see them. |
| `test_memory.py` | Free-text append/edit (no ids/stamps — `old_string`/`new_string`/`replace_all`), verbatim storage, entry-count ignores headings, persona form + relationship-memory preservation, raw save round-trip + 400/404, onboarding endpoint (LLM-structured multi-file distribution + name-only no-LLM path). |
| `test_words.py` | Add/list, 409 duplicate, 400/422 invalid, manual adjust + clamping, events, delete, personal note (PUT set/trim/clear + 404) and its appearance in the chat prompt's target-words block. |
| `test_scoring.py` | Matrix deltas, zero floor, daily cap (+manual bypass), times_used/last_used semantics, decay (idle threshold, idempotent), spaced-repetition picker ordering. Words created via `repo.insert_word(Word(...))`; scoring fns called WITHOUT a db arg. |
| `test_mongo_connection.py` | `@pytest.mark.live` connection smoke: `mongo.ping()`, write/read/delete round-trip, `ensure_indexes()` creates `uq_user_text`. Deselected by default (needs real Atlas). |
| `test_conversations.py` | Create picks 3 targets, mocked topics, category patch, CRUD, search: bm25 window, regex, full_conversation, empty. |
| `test_chat.py` | Basic turn, 400/404, tool loop + stored tool_calls transparency (`memory_update` action='append'), `memory_update` tolerates a `.md` suffix on `file`, GET .../messages returns persisted `word_events` on the user message (with word_text), history reconstruction (AIMessage+ToolMessage pair with original id), judge events applied, judge failure swallowed, auto-title, agent adjust_word_score (+unknown id), opener (+400 when non-empty). |
| `test_dashboard.py` | Health, stats aggregation. |
| `test_auth.py` | Google OAuth + session (Google network calls mocked at `auth_service`, real signed cookies). Login 307→Google with correct params; callback happy path creates the user; rejects missing/mismatched state and unverified email; `/me` 401 without session; logout clears the cookie; **first user adopts `default` data** (second user does not); **data-isolation guarantee** — user A's words invisible to B and vice-versa, A's survive. Uses `anon_client`. |
| `test_model.py` | BYO-key model config: tier catalogue (swift/sage + model ids), status (reports configured, NEVER returns the key), set-key (verify mocked → encrypts+stores, asserts plaintext NOT in DB & decrypts back; rejects bad key/unknown tier/empty), tier switch (+unknown, +requires existing key), the LLM-route gate (403 when no model). `verify_key` mocked — no real Gemini traffic. |
| `test_settings.py` | Hard-delete endpoints: purge conversations (keeps words/memories), purge memories (keeps persona/words/conversations), purge-all (keep_words true/false — full wipe asserts `cleared_model` + key/tier cleared; keep-words asserts model config SURVIVES), idempotency. |
| `test_live_smoke.py` | `@pytest.mark.live`: real enrichment, real chat turn + auto-title, real judge scoring, real topics. Costs quota. |
| `last_report.txt` | Latest run_tests.py report (generated). |

Patterns: patch LLM factories on the CONSUMING module (`chat_service.get_chat_model`, `judge_service.get_judge_model`, `topic_service.get_utility_model`) — and `resolve_for_user` on the same modules (already done in `mock_llms`). NEVER hardcode any key (Gemini/Fernet/etc.) in a test file — `ENCRYPTION_KEY` is read from `.env`; use `crypto_service.encrypt(...)` when a test needs stored ciphertext. Ids are ObjectId hex STRINGS — for "absent id" 404 checks use a valid-but-unused ObjectId string like `"000000000000000000000000"` (a bare int/`"99999"` isn't a valid ObjectId). Every new API/feature: happy path + error paths + side effects, same change.
